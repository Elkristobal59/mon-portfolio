Assistant Documentaire RAG

Un système de question-réponse sur corpus documentaire, capable d'interroger 500+ documents PDF avec une précision sémantique grâce à LangChain, ChromaDB et l'API Gemini.

Python LangChain ChromaDB Gemini API Vector DB RAG

🎯 Contexte & Objectif

Dans le cadre de ma formation Data Science à Jedha, j'ai conçu un assistant documentaire intelligent capable de répondre à des questions en langage naturel sur un corpus de documents internes. Le défi : rendre un volume important de PDF "interrogeables" sans hallucinations, en s'appuyant sur les passages réellement présents dans les documents.

🏗️ Architecture du Pipeline RAG

  1. Ingestion & Chunking

    Chaque PDF est découpé en chunks de 512 tokens avec chevauchement de 64 tokens pour conserver le contexte entre blocs.

  2. Vectorisation (Embeddings)

    Chaque chunk est transformé en vecteur dense via le modèle d'embeddings de Gemini. Les vecteurs sont stockés dans ChromaDB.

  3. Recherche Sémantique

    À la question de l'utilisateur, on calcule son vecteur et on retrouve les k chunks les plus proches par similarité cosinus.

  4. Génération Augmentée (LLM)

    Les chunks sélectionnés servent de contexte au prompt envoyé à Gemini, qui génère une réponse ancrée dans les sources.

💡 Apprentissages Clés

  • ✓ La taille des chunks est un paramètre critique : trop petits → perte de contexte, trop grands → bruit dans le prompt.
  • ✓ Le reranking des chunks avant injection améliore significativement la qualité des réponses.
  • ✓ Afficher les sources dans l'UI est essentiel pour établir la confiance utilisateur (éviter les hallucinations silencieuses).
